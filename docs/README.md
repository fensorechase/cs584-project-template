# Documentation

## Project Documentation

This folder contains all project documentation:

- **SOW.md** - Statement of Work (requirements, scope, timeline)
- **architecture.md** - System architecture and design decisions
- **user-guide.md** - User manual for your GH partner
- **dev-guide.md** - Developer documentation
- **meeting-notes/** - Notes from meetings with GH partner

## Documentation Guidelines

### For Non-Technical Readers (Your GH Partner)
- Use simple language
- Include screenshots
- Focus on "what" and "why", not "how"

### For Developers
- Explain technical decisions
- Include setup instructions
- Document APIs and data models
